plugin配置

示例代码如下

public class DemoConfig extends JFinalConfig {
  public void configPlugin(Plugins me) {
  DruidPlugin dp = new DruidPlugin("jdbc:mysql://localhost/db_name", "userName", "password");
    me.add(dp);
    ActiveRecordPlugin arp = new ActiveRecordPlugin(dp);
    me.add(arp);
    arp.addMapping("user", User.class);
    arp.addMapping("article", "article_id", Article.class);
  }
}

改代码的配置，url应该改为自己的数据库，列如有数据库database,则url为jdbc:mysql://localhost：3306/database. 用户名与密码也要改为自己的，关于代码

    arp.addMapping("article", "article_id", Article.class);```
如果你的表的主键id为xx_id,则使用arp.addMapping("article", "article_id", Article.class)，article为表名，
如果你的主键字段就为id，则使用arp.addMapping("user", User.class)，**重点，这两句代码不要一起写入，**

# Model
官网解释： Model是ActiveRecord中最重要的组件之一，它充当MVC模式中的Model部分，
根据官网以及自己的调试，model中为我们提供了放便的增删改查，也就是我们不必去自己实现，只需要调用model中的方法即可完成数据库的增删改查。
官网model示例代码
```java
public class User extends Model<User> {
    public static final User dao = new User().dao();
}

关于该处，不建议这样写，一般在service层中new 出dao，然后就是该处博主我也有很多地方没有清楚，正在研究中，有兴趣的同学可以自己去官网查看，询问。
